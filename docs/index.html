01. What xi is

Minimal surface area. Maximum accountability.

Agent runtime

xi runs as a terminal-native agent with model provider abstraction, deterministic tool wiring, and explicit session boundaries.

End-to-end observability

Each tool call is stored with timing, params, output, and errors. No black box behavior. No lost context.

Production-minded by default

The architecture enforces controlled writes via AgentFS while allowing sandboxed execution through Just Bash.

02. Why it's different

Concrete differences from typical coding agents

Every action is auditable

Reads, writes, edits, shell commands, and errors are persisted in a single SQLite timeline.

Session = portable artifact

Each session becomes one `.db` file under `.xi/sessions/` you can inspect, archive, and diff.

Safe write path

File writes flow through AgentFS SDK instead of ad-hoc shell mutation, reducing accidental drift.

Provider flexibility

Switch providers through a unified interface using the Vercel AI SDK foundation.

Terminal-first UX

Built for engineers who live in the CLI and want clear execution semantics, not hidden automation.

Trackable by design

Auditability isn't an add-on telemetry product. It's a first-class architectural primitive.
